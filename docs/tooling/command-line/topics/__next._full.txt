{% info title="Before proceeding" %}
Ensure you [**install**](/docs/tooling/command-line/installation#getting-started) the CLI, [**log in**](/docs/tooling/command-line/installation#login) to your Clikkle account, and [**initialize**](/docs/tooling/command-line/installation#initialization) your Clikkle project.
{% /info %}


The Clikkle CLI can create, update, delete, and get topics, as well as configure the provider and the subscribers.

# Initialize topic {% #initialize-topic %}

Create a new topic using the following command:

```
clikkle init topics
```

# Pull topics {% #pull-topics %}

You can also pull your existing Clikkle topics from the Clikkle Console using the `pull` command in the folder containing your `clikkle.config.json` file.

```sh
clikkle pull topics
```

# clikkle.config.json {% #clikklejson %}

After [initializing](/docs/tooling/command-line/installation#initialization) your Clikkle project and pulling your existing topics, your `clikkle.config.json` file should look similar to the following:

```json
{
    "projectId": "<PROJECT_ID>",
    "endpoint": "https://<REGION>.cloud.clikkle.io/v1",
    "topics": [
        {
            "$id": "N1IzUIJiOicGbhJye",
            "$createdAt": "2024-07-01T14:40:43.381+00:00",
            "$updatedAt": "2024-07-01T14:40:43.381+00:00",
            "name": "Anime",
            "emailTotal": 3,
            "smsTotal": 0,
            "pushTotal": 0,
            "subscribe": [
                "users"
            ]
        },
        {
            "$id": "eyJhbGciOiJIUzI1N",
            "$createdAt": "2024-07-01T14:41:19.029+00:00",
            "$updatedAt": "2024-07-01T14:41:28.751+00:00",
            "name": "Music",
            "emailTotal": 2,
            "smsTotal": 0,
            "pushTotal": 0,
            "subscribe": [
                "users",
                "any"
            ]
        }
    ]
}
```

# Push topics {% #push-topics %}

Use the `push` command in the folder containing your `clikkle.config.json` file to push the changes you made.


```sh
clikkle push topics
```

# Commands {% #commands %}

The messaging command allows you to send, get, update, and delete push notifications, SMS text messages, and emails. You can create a new provider like Mailgun and SendGrid and create, update, get, and delete topics and subscribers. Clikkle messaging CLI commands generally follow the following syntax:
```sh
clikkle messaging [COMMAND] [OPTIONS]

{% info title="Before proceeding" %}
Ensure you [**install**](/docs/tooling/command-line/installation#getting-started) the CLI, [**log in**](/docs/tooling/command-line/installation#login) to your Clikkle account, and [**initialize**](/docs/tooling/command-line/installation#initialization) your Clikkle project.
{% /info %}


The Clikkle CLI can create, update, delete, and get topics, as well as configure the provider and the subscribers.

# Initialize topic {% #initialize-topic %}

Create a new topic using the following command:

```
clikkle init topics
```

# Pull topics {% #pull-topics %}

You can also pull your existing Clikkle topics from the Clikkle Console using the `pull` command in the folder containing your `clikkle.config.json` file.

```sh
clikkle pull topics
```

# clikkle.config.json {% #clikklejson %}

After [initializing](/docs/tooling/command-line/installation#initialization) your Clikkle project and pulling your existing topics, your `clikkle.config.json` file should look similar to the following:

```json
{
    "projectId": "<PROJECT_ID>",
    "endpoint": "https://<REGION>.cloud.clikkle.io/v1",
    "topics": [
        {
            "$id": "N1IzUIJiOicGbhJye",
            "$createdAt": "2024-07-01T14:40:43.381+00:00",
            "$updatedAt": "2024-07-01T14:40:43.381+00:00",

                "users"

                "users",
                "any"
            ]
        }
    ]
}
```

# Push topics {% #push-topics %}

Use the `push` command in the folder containing your `clikkle.config.json` file to push the changes you made.


```sh
clikkle push topics
```
