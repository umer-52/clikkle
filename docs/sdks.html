SDKs

Clikkle provides SDK libraries for major programming languages and platforms so you don't have to write code for interacting with our API protocols from scratch.

We're always working on improving and extending the current stack of available platforms and SDKs, listed below is a list of official libraries the Clikkle team is maintaining.

Client

Client libraries for integrating with Clikkle to build client-based applications and websites. Read one of the many quick starts guides for your framework of choice to start building your first application.

Protocols

We are always looking to add new SDKs to our platform. If the SDK you are looking for is still missing, labeled as beta or experimental, or you simply do not want to integrate with an SDK, you can always integrate with Clikkle directly using any standard HTTP, GraphQL, or WebSocket clients and the relevant Clikkle protocol.

Clikkle supports multiple API protocols for maximum flexibility and developer convenience. You can learn more about how to integrate directly with them using one of the following available guides:

Utility classes

Clikkle's SDKs provide useful utility classes to make your development experience easier. Use these classes and methods to reduce guess work and get better code suggestions in your IDE.

IDs

Clikkle has utility classes to help you handle IDs. These generate the correct ID format to be passed to the Clikkle APIs.

  • Copy
  • Copy

Clikkle generates unique IDs that are both random and ordered. Clikkle IDs combine a timestamp prefix based on microseconds and a random UID string, formatted as <timestamp_hex><uid_hex>.

Enums

Clikkle has enumeration classes for predefined strings used different parameters used for the Clikkle APIs.

These enums are available for authenticator type, name, OAuth provider, password hash types, browsers, authentication factors, index types, credit cards, image gravity, image format, relationship types, SMTP encryption, Function runtime, messaging provider type, compression algorithms, execution methods, and country flags.

You can discover the available enums in each SDK at the source.
