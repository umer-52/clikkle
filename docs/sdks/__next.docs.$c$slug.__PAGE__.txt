We're always working on improving and extending the current stack of available platforms and SDKs, listed below is a list of official libraries the Clikkle team is maintaining.

# Utility classes {% #utility-classes %}
Clikkle's SDKs provide useful utility classes to make your development experience easier.
Use these classes and methods to reduce guess work and get better code suggestions in your IDE.

## IDs {% #ids %}
Clikkle has utility classes to help you handle IDs. 
These generate the correct ID format to be passed to the Clikkle APIs.

Clikkle generates unique IDs that are both **random** and **ordered**. 
Clikkle IDs combine a timestamp prefix based on microseconds and a random UID string, formatted as `<timestamp_hex><uid_hex>`.

## Enums {% #enums %}
Clikkle has enumeration classes for predefined strings used different parameters used for the Clikkle APIs.

These enums are available for authenticator type, name, OAuth provider, 
password hash types, browsers, authentication factors, index types, credit cards, 
image gravity, image format, relationship types, SMTP encryption, Function runtime, 
messaging provider type, compression algorithms, execution methods, and country flags.

These generate the correct ID format to be passed to the Clikkle APIs.

Clikkle generates unique IDs that are both **random** and **ordered**. 
Clikkle IDs combine a timestamp prefix based on microseconds and a random UID string, formatted as `<timestamp_hex><uid_hex>`.

image gravity, image format, relationship types, SMTP encryption, Function runtime, 
messaging provider type, compression algorithms, execution methods, and country flags.

## Queries and permissions {% #queries-and-permissions %}
Clikkle has utility classes for queries and permissions. 
You can learn more about [query utility classes](/docs/products/databases/queries) and [permissions utility classes](/docs/advanced/platform/permissions) in their own pages.

## File I/O {% #file-io %}
Depending on your platform, you will also need some helpers to interface with system I/O.
Learn more about [storage input file classes](/docs/products/storage/upload-download#input-file).
